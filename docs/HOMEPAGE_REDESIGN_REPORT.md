## 7. 발견 사항 / 단순화 제안

### 명세에 없어 결정한 사항 (보고만)
1. **connect 섹션** — 새 흐름에서 빠짐. 삭제 vs 보존 갈림. **HTML 주석화**로 코드 보존(복원 한 줄). 이전 BACnet/RS-485 연동 디테일은 영업 자료로 가치 있으므로 비공개 보존.
2. **kpi-strip 통째 삭제** — 라이브 카드 그리드가 KPI 역할 일부 흡수. 15~25%/0원/Read-Only 류 텍스트는 라이브 카드 + 히어로 viz-meta로 분산.
3. **카피 신뢰 강화** — 사용자 명세에 없던 "수개월간 발견되지 않음" 같은 펀치라인은 기존 카피 톤 유지하면서 "방치" 강조로 통일.

### 다음 단계 후보
1. **GitHub Actions로 build 검증** — 현재 raw HTML이라 빌드 없음. 향후 HTMLHint/lint 워크플로 추가 시 PR 시점 오류 조기 발견.
2. **모바일 우선 추가 미세 조정** — 현재 768px 미만 단일 미디어쿼리(960px)만 사용. 480px 대응을 별도 미디어쿼리로 분리하면 더 견고.
3. **이미지 사용 시 `loading="lazy"`** — 현재 이미지 0. 향후 현장 사진 추가 시 lazy load.
4. **Open Graph 메타 태그** — `<meta property="og:title">`, `og:image` 등이 없음. 카카오톡/Slack 공유 시 미리보기 빈약. 추가 검토.
5. **요금 옵션 카드의 "기설치 전력미터 교차검증" 문구** — 기존 가격표의 함정 문구를 보존했지만 옵션 카드 1줄로 압축. 영업 시 동일하게 안내되는지 확인 권장.

## 한 줄 회고 — 오늘 배운 것 (비개발자 오너용)

오늘 배운 것 1 — **"카피 한 줄 바꾸는 일은 사실 메시지 위계 바꾸는 일"**: 가격 숫자 4단을 제거한 건 코드 100줄 들어내는 일이지만, 진짜 변화는 "우리는 가격이 아니라 효과로 설득한다"는 영업 자세의 변경이다. 코드는 그걸 따라간 결과일 뿐. **카피가 먼저 결정되고 코드가 그걸 구현한다**.

오늘 배운 것 2 — **"단계별 커밋이 보험이다"**: 큰 변경을 한 commit으로 묶으면 문제 발생 시 통째로 되돌려야 한다. C1·C2+C3·C4로 3단 분리한 덕에 "C4 정리만 되돌리고 본문은 살리기" 같은 정교한 롤백이 가능. **Git commit은 코드 변경 단위가 아니라 "되돌릴 수 있는 단위"로 끊는다**. (대비: 한 번에 다 했다면 "데드 CSS 정리 중 실수 → 본문도 같이 사라짐" 위험)
