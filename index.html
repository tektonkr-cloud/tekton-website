The Problem

센서가 거짓말을 합니다.
그래도 화면은 '정상'이라 적힙니다.

고장은 매달 일어나지만, 모르면 그대로 방치됩니다.
금액보다 무서운 건 '몰랐다'는 사실입니다.

01 / 센서 드리프트

온도센서가
5°C 높게 표시됩니다.

AHU 급기 덕트 온도센서 경년열화. 실제 22°C → 센서 27°C 표시. 냉방 밸브 과다 개방 중. BAS 화면엔 "정상".

↑ 고장을 모른 채 방치
02 / 통신 두절

60개 포인트가
수개월째 고착입니다.

RS-485 배선 노후, 접촉 불량. 센서 60개 데이터 고착. 서버엔 마지막 정상값만 표시. 평균 2~6개월 뒤 우연히 발견.

↑ 수개월간 발견되지 않음
03 / 밸브 고착

DDC는 닫혔다 하지만
실제론 열려 있습니다.

FCU 냉온수 밸브 액추에이터 고착. DDC 닫힘(0%) 명령 → 실제 30% 열림. 겨울철 냉수 지속 유입.

↑ 모르고 계속 운전

T-ARCH는

직접 고치지 않습니다. 무엇을·언제·왜 봐야 하는지 짚어드립니다.

How it works

4단계 포렌식이
매 초 데이터를 의심합니다.

센서가 정상이라 말해도 다시 확인합니다. 사람의 발 대신 데이터를 검증합니다.

STEP 01

센서 무결성 검증

고착·급변·드리프트·범위 이탈을 실시간 탐지합니다.

STEP 02

통신 건전성 분석

RS-485 패킷 손실, 타임스탬프 불일치, 데이터 고착을 자동 감지합니다.

STEP 03

에너지 이상패턴

동시 냉난방, 비가동 시간 운전, 낭비 금액을 원 단위로 정량화합니다.

STEP 04

제어 명령 검증

DDC 명령값 vs 실측값 불일치, 인터록 위반을 자동 탐지합니다.

BACnet / IP
REST API 연동

BACnet/IP 네트워크가 구성된 현장에서 가장 빠른 방식입니다. 물리 배선 없이 아래 세 가지 중 하나만 제공하시면 됩니다.

  • ARead-Only 계정 생성 후 IP/포트 전달
  • BDDC 포인트 맵(Object List) 파일 제공
  • CREST API 엔드포인트 오픈 (IP + 포트 허용)
  • 4T-ARCH 자동 탐지 → 포렌식 시작
